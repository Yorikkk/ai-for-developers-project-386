#### Описание
Описание проекта
«Запись на звонок» — это упрощенный сервис бронирования времени по мотивам Cal.com.

Cal.com — это сервис, где пользователь публикует доступные интервалы, а другой человек выбирает свободное время и бронирует встречу. У текущего приложения тот же базовый сценарий: владелец публикует доступное время для встреч, а другой пользователь выбирает свободный слот и записывается на звонок.

В проекте нет авторизации, личных кабинетов и интеграций с внешними календарями. Функциональность намеренно ограничена, чтобы сосредоточиться на главном.

В результате должно получиться небольшое, но законченное приложение. Пользователь видит свободные слоты по 30 минут, может выбрать время и оформить запись, а владелец календаря — посмотреть список предстоящих встреч. Этого достаточно, чтобы пройти весь путь от проектирования до запуска сервиса в облаке.

## Запуск приложения

### Разработка (Development)

Для локальной разработки запускайте фронтенд и бэкенд отдельно:

```bash
# Запуск бэкенда (порт 3000)
make backend-dev
# или
cd backend && npm run dev

# Запуск фронтенда (порт 5173)
make frontend-dev
# или
cd frontend && npm run dev
```

### Production с Docker

Приложение можно запустить в Docker-контейнере, где фронтенд и бэкенд объединены в один образ.

#### Быстрый старт с docker-compose

```bash
# 1. Создать файл .env из примера (необязательно)
cp .env.example .env

# 2. Собрать и запустить контейнер
make docker-compose-up
# или
docker-compose up -d

# 3. Открыть приложение в браузере
# http://localhost:3000
```

#### Использование Docker напрямую

```bash
# Сборка образа
make docker-build
# или
docker build -t booking-app .

# Запуск контейнера
docker run -p 3000:3000 -e PORT=3000 booking-app

# Запуск с другим портом
docker run -p 8080:8080 -e PORT=8080 booking-app
```

#### Доступные команды Makefile

```bash
# Development
make frontend-dev           # Запуск фронтенда в dev режиме
make backend-dev            # Запуск бэкенда в dev режиме
make frontend-test          # Запуск E2E тестов

# Docker
make docker-build           # Собрать Docker-образ
make docker-run             # Запустить контейнер
make docker-compose-up      # Запустить через docker-compose
make docker-compose-down    # Остановить docker-compose
make docker-compose-logs    # Просмотр логов
make docker-compose-build   # Пересобрать и запустить
```

#### Переменные окружения

| Переменная | Описание | Значение по умолчанию |
|-----------|----------|----------------------|
| `PORT` | Порт для запуска приложения | `3000` |
| `NODE_ENV` | Окружение (production/development) | `production` |

#### Архитектура Docker-образа

- **Многоэтапная сборка**: уменьшает размер финального образа
- **Этап 1**: Сборка фронтенда (React + Vite)
- **Этап 2**: Компиляция бэкенда (TypeScript)
- **Этап 3**: Финальный образ с объединенным приложением
- **Статические файлы**: бэкенд обслуживает собранный фронтенд из папки `public`
- **SPA routing**: все неизвестные маршруты перенаправляются на `index.html`

#### Проверка работы

После запуска контейнера:

1. Откройте браузер: `http://localhost:3000`
2. Проверьте API: `http://localhost:3000/event-types`
3. Убедитесь, что доступны все страницы:
   - Главная: `/`
   - Каталог событий: `/book`
   - Бронирование: `/book/:eventTypeId`
