#### Агент:
SourceCraft Code Asistant

#### последовательность создания проекта
1. Создал файл AGENTS.md в котором описал доменные сущности
    ```
    Это новый проект
    Язык общения русский

    Он будет содержать следующие доменные сущности:
    - Владелец
    - Тип события
    - Слот
    - Бронирование
    - Публичный сценарий гостя

    Доменная сущность "владелец" содержит следующие поля
    - id
    - Имя

    Доменная сущность "тип события" содержит следующие поля
    - id
    - название
    - длительность события в минутах

    Доменная сущность "слот" содержит следующие поля
    - id
    - ссылка на тип события
    - ссылка на владельца
    - ссылка на публичный сценарий гостя
    - дата и время в часах и минутах

    Доменная сущность "бронирование" содержит следующие поля
    - id
    - дата и время в часах и минутах
    - ссылка на тип события
    - ссылка на владельца
    - ссылка на публичный сценарий гостя

    Доменная сущность "публичный сценарий гостя" содержит следующие поля
    - id
    - имя
    ```

    Кстати на русском он так и не заговорил
2. Запустил агента со следующей задачей
    ```
    Необходимо подготовить TypeSpec-спецификацию, со следующими API-контрактами
    - Создание типа события
    - Получение списка типов события
    - Получение списка слотов
    - Создание бронирования
    - Получение списка бронирования
    ```

    Он создал файл спецификации main.tsp, потом поставил пакеты для проверки спецификации, несколько раз запускал проверку, менял и правил там что-то и наконец успокоился.
    В итоге еще появилась папка generated с OpenAPI-спецификацией
3. Создал папку для фронтенда следующей задачей
    ```
    Добавь фронтенд для проекта в отдельную папку ./frontend
    Настрой его для использования следующего стека
    TypeScript, Vite, Mantine, Prism
    ```
4. Очистил AGENTS.md и запустил процесс его переформирования командой ./init
    Агент создал описание для AGENTS.md, а также добавил новую папку ./codeassistant с следующими папками и файлами AGENTS.md в них
    - ./codeassistant/rules-architect
    - ./codeassistant/rules-ask
    - ./codeassistant/rules-code
    - ./codeassistant/rules-debug
