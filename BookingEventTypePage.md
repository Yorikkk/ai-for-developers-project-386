Дизайн как на картинке в файле @/design/03-book-event-type.png

Для АПИ можно использовать эмулятор АПИ Prism

На страницу передается ID типа события из списка EventType
Тип события выбирается из АПИ ./api/event-types/{eventTypeId}, где eventTypeId - :eventTypeId из роута

Описание:
1. Header - такой же как и в HomePage
2. Content - представляет собой заголовок и три блока с background-color: белый
  Заголовок - представляет собой заголовок крупного размера
    Текст заголовка: Встреча X минут, где X - durationMinutes из выбранного типа события
  Блоки расположены слева направо
  Левый блок - представляет собой карточку с заголовком и тэгом справа от заголовка
    Текст заголовка: Встреча X минут, где X - durationMinutes из выбранного типа события
    Текст тэга: X мин, где X - durationMinutes из выбранного типа события
    Под заголовком располагается подпись с текстом: text из выбранного типа события
    Ниже располагается карточка с выбранной датой из календаря центрального блока
        Карточка представляет собой блок с заголовком и подписью. Фон блока - светло-серый, края закруглены.
        Заголовок блока - стандартный размер, цвет - серый, текст - Выбранная дата
        Подпись блока - стандартный размер, цвет - черный, текст - дата из календаря, которую выбрали кликом из центрального блока
    Ниже располагаются карточки забронированных событий на выбранную дату из АПИ ./api/bookings?date={date}, где date - дата которую выбрали кликом из календаря центрального блока
        Карточка представляет собой блок с заголовком и подписью. Фон блока - светло-серый, края закруглены.
        Заголовок блока - стандартный размер, цвет - серый, текст - Выбранное время
        Подпись блока - стандартный размер, цвет - черный, текст - utcDateTime из АПИ ./api/bookings/?date={date}, где date - дата которую выбрали кликом из  календаря центрального блока
  Центральный блок - представляет собой календарь с переключением месяцев и возможностью клика на дату
  Правый блок - представляет собой заголовок "Статус слотов" и список временных промежутков, которые можно выбрать кликом. Список промежутков забирается из АПИ ./api/slots/?eventTypeId={typeId}, где typeId - id типа события. Каждый промежуток представляет собой блок с текстом слева и подписью справа. Фон блока - белвыйй, края закруглены, border - 1px, серый.
    Текст слева - "{time_from} - {time_to}", где time_from - время в формате hh:mm, берется из dateTime модели Slot, time_to - время в формате hh:mm, считается как dateTime модели Slot + durationMinutes модели EventType
    Текст справа - может иметь два варианта: "Свободно" и "Занято". Алгоритм формирования следующий - если в промежуток между time_to и time_from не попадает ни один utcDateTime из АПИ ./api/bookings/?date={date}, то пишем "Свободно", если попадает, то "Занято".
    Если слот занят, то фон блока светло-серый и цвет текста внутри блока серый
