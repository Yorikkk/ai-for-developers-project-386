# E2E Тесты для страницы каталога бронирования

## Обзор
Этот каталог содержит E2E тесты, написанные с использованием Playwright для проверки функциональности страницы каталога бронирования.

## Структура
```
tests/
├── booking-catalog/
│   ├── booking-catalog.spec.ts     # Основные тесты страницы
│   └── api-integration.spec.ts     # Тесты интеграции с API
└── helpers/
    └── test-utils.ts               # Вспомогательные функции
```

## Запуск тестов

### Предварительные требования
Убедитесь, что фронтенд и бэкенд запущены:
```bash
# Терминал 1: Бэкенд
cd backend && npm run dev

# Терминал 2: Фронтенд
cd frontend && npm run dev
```

### Команды запуска
```bash
# Запуск всех тестов
npm run test:e2e

# Запуск с UI для отладки
npm run test:e2e:ui

# Запуск в debug режиме
npm run test:e2e:debug

# Запуск с видимым браузером
npm run test:e2e:headed

# Запуск только в Chromium
npm run test:e2e:chromium
```

## Тестовые сценарии

### booking-catalog.spec.ts
1. **Базовая загрузка страницы** - проверка отображения заголовка и описания
2. **Отображение карточек** - проверка наличия карточек типов событий
3. **Навигация** - проверка перехода на страницу выбора слотов
4. **Состояние загрузки** - проверка отображения скелетонов

### api-integration.spec.ts
1. **Fallback при ошибке** - проверка использования fallback данных при ошибке API
2. **Успешный запрос** - проверка отображения данных из API
3. **Пустой ответ** - проверка обработки пустого массива от API

## Результаты
После запуска тестов отчеты доступны в:
- HTML отчет: `playwright-report/index.html`
- Скриншоты: `test-results/` (только при падении тестов)
